## 问题分析

### 日志对比

**正常收到的文本消息（14:37 之前）：**
```
from:zackwang type:text content:hello
from:zackwang type:text content:你好，测试
from:zackwang type:text content:谢谢你
from:zackwang type:text content:你是谁谁
```

**之后收到的消息（14:37 之后）：**
```
from:sys type:event content:
from:sys type:event content:
from:sys type:event content:
...
```

### 根本原因

**个人微信 → 企业成员 的消息，走的是"微信客服"事件通道，不是"应用文本消息"通道。**

企业微信推送的是 `MsgType=event`，不是 `MsgType=text`。

当前代码只处理文本消息：
```python
if mt == "text" and ct:
    # 保存并回复
```

事件消息（`mt == "event"`）被忽略了。

---

## 解决方案

### 方案 A：处理事件消息（推荐）

修改 `main.py` 的 `handle_message` 函数，识别并处理事件消息：

```python
# 事件类型判断
ev_type = msg.findtext("Event", "")
if ev_type == "enter_agent":
    # 用户进入应用
    pass
elif ev_type == "send_msg":
    # 微信客服消息事件
    # 需要额外调用 API 获取消息内容
    pass
```

### 方案 B：启用微信客服功能

1. 企业微信后台 → 应用管理 → 微信客服
2. 添加客服账号，绑定到应用
3. 配置客服账号的回调 URL

### 方案 C：通过应用发消息

让客户通过企业微信应用（不是直接聊天）发消息，这样会走文本消息通道。
